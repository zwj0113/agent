步骤 1：生成 OpenSpec (接口契约)

指令内容：

作为架构师，请按照 OpenAPI 3.0 规范为我们的 AI Agent MVP 编写一份 api_spec.yaml。

MVP 需求范围：

POST /api/v1/chat/stream: 接收用户消息，返回 Server-Sent Events (SSE) 流。

SSE 数据流必须包含三种类型：

thought: Agent 的思考过程。

call: 工具调用的名称和参数。

answer: 最终生成的文本内容。

GET /api/v1/health: 基础健康检查。

请确保 YAML 定义清晰，尤其是 SSE 的 data 结构，以便前后端严格对齐。

步骤 2：后端基础骨架 (FastAPI + LangChain)

指令内容：

请阅读 api_spec.yaml，使用 Python 3.10+ 和 FastAPI 实现后端基础代码。

技术要求：

使用 langchain 和 langchain-openai 创建一个简单的 ReAct Agent。

实现一个自定义的 AsyncCallbackHandler，捕获 on_llm_thought (或类似逻辑)、on_tool_start 和 on_llm_new_token 事件。

将这些事件封装成符合规范的 SSE 格式返回。

先内置一个简单的工具（例如 get_weather 或 calculator）用于演示工具调用过程。

暂时使用内存存储会话历史。

请输出完整的 main.py 和必要的依赖配置文件 requirements.txt。

步骤 3：前端交互骨架 (React + Tailwind)

指令内容：

请根据 api_spec.yaml 定义的 SSE 结构，使用 React (Next.js) 和 Tailwind CSS 编写聊天界面。

功能要求：

聊天窗口布局：左侧消息历史，底部输入框。

消息流处理：使用 fetch 或 EventSource 监听 /api/v1/chat/stream。

实时展示逻辑：

当收到 thought 类型时，在界面显示一个“思考中...”的半透明卡片。

当收到 call 类型时，显示一个“正在调用工具 [名称]”的标签。

当收到 answer 类型时，使用打字机效果实时渲染 Markdown 文本。

确保界面响应式，风格类似 Claude.ai。

请输出单个文件的 ChatApp.tsx 或完整的组件代码。

步骤 4：联调与校验

指令内容：

现在前后端代码已生成。请你扮演 QA 工程师，检查 main.py 的 SSE 输出格式是否与 api_spec.yaml 完全一致。

特别检查：

跨域 (CORS) 配置是否已在 FastAPI 中开启，以允许前端访问。

异步生成器 (Async Generator) 是否正确处理了连接中断的情况。

如果一致，请提供一个 curl 命令用于测试接口。
